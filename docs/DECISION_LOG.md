# Phase 4 决策补充（2026-07-13）

- D-041 已确认：正文正式格式采用 Markdown，Web 不执行原始 HTML。
- D-042 已确认：本阶段只落地 Post、Tag、PostTag；评论、互动、通知及 AI/RAG 字段延期。
- D-043 已确认：slug 使用标题摘要和加密随机后缀，支持中文及相同标题。

# Phase 3B 决策（2026-07-13）

- D-044 已确认：正式事务邮件单一使用 Resend HTTP API；支持自定义域名发件人、HTML/文本、明确错误、幂等键及后续投递/退信/投诉事件。
- D-045 已确认：重发时原子失效全部未使用旧 Token，再创建新 Token；任何发送失败都不得伪装成功。
- D-046 已确认：生产邮件配置缺失时启动失败，禁止回退 DevelopmentMailService。

# Phase 5 决策（2026-07-13）

- D-047 已确认：评论只保存纯文本，最多 5,000 字符，不扩展 Markdown、图片或附件渲染面。
- D-048 已确认：评论数据只形成两层；回复二级回复时归一到一级父评论，以 `replyToUserId` 保留语义。
- D-049 已确认：作者与 ADMIN 可幂等软删除评论；保留作者和回复结构，正文公开输出改为删除占位；MODERATOR 权限不扩大。
- D-050 已确认：评论与单条站内通知在同一 Prisma 事务中创建；自通知不生成，每次评论最多一名接收人。
- D-051 已确认：通知采用数据库 `readAt`，页面加载时读取，不引入 Redis、轮询、WebSocket、SSE 或邮件通知。
- D-052 已确认：本阶段依赖前端禁用提交降低重复点击；未实现持久化 Idempotency-Key，网络不确定性下的人工重试重复风险作为后续技术债。

# Community UI V1 决策（2026-07-13）

- D-053 已确认：桌面首页采用固定 Header + 固定 Sidebar + 原创静态像素 Hero + 分类栏 + 真实帖子 Grid；参考图只提供布局和密度，不复制其品牌、人物、帖子或游戏素材。

- D-056 已确认：尚未实现的搜索、消息、热门、收藏、点赞、历史和设置入口必须只读、禁用或明确标识“即将开放”；“我的发布”和“我的通知”连接真实路由。
- D-057 已确认：桌面栅格以 1280px 三列、1440px 和 1920px 四列验收；移动端不在本轮新增专项导航或设计范围。

# 产品重新定位决策（2026-07-14）

- D-065 已确认：Liftoff 的唯一正式定位从“AI Builder / AI 开发者社区”调整为“AI 创业 / AI 副业赚钱社区”；技术讨论降为第二层支撑内容。
- D-066 已确认：正式 Slogan 使用 `Find. Build. Earn. Repeat.`，取代 D-002 的 `Build. Ship. Repeat.`。
- D-067 已确认：目标用户优先级为想利用 AI 做副业的普通用户、独立开发者、AI 创业者、AI 工程师；首页和内容验证不得反向排序。
- D-068 已确认：社区内容闭环为“赚钱机会 → 项目验证 → 项目拆解 → AI Coding → 技术优化 → 获客与变现 → 商业复盘 → 新机会”。
- D-069 已确认：第一层栏目为赚钱机会、副业项目、项目验证、项目拆解、获客与变现、收入案例、失败复盘和一人公司；Vibe Coding、Agent、RAG、MCP、FDE、OPC、工具与资源属于第二层。
- D-070 已确认：本轮只重构文档。现有代码、UI、API、Prisma、数据库、认证、帖子、评论和通知实现不变；后续改造必须另行授权。
- D-071 已确认：收入内容不得承诺收益或以截图作为唯一证明；正式产品化前必须确定真实性、隐私、审核和风险提示规则。
