# Liftoff Validation MVP 产品需求

## 1. 文档目标

本文定义 Liftoff 从 Phase 2 开始的 Validation MVP。它只描述需要验证核心社区行为链的产品范围，不代表相关业务已经实现。

详细验证策略见 [VALIDATION_MVP.md](VALIDATION_MVP.md)。

产品内容定位以 [PRODUCT_VISION.md](PRODUCT_VISION.md) 为准：Liftoff 是 AI 创业 / AI 副业赚钱社区，技术讨论是支撑项目实现和商业结果的第二层内容。

## 2. 核心目标

验证目标用户是否愿意完成：

```text
访问首页 → 浏览帖子 → 邮箱注册 → 发帖 → 回复 → 通知 → 回访
```

在该链稳定运行前，不继续扩展社交、AI、游戏或复杂运营能力。

同时验证内容价值链是否成立：

```text
发现赚钱机会 → 验证项目 → 项目拆解 → 做出产品 → 获客变现 → 商业复盘
```

## 3. Validation MVP 角色

| 角色     | 最小能力                                               |
| -------- | ------------------------------------------------------ |
| 访客     | 浏览首页、帖子和简单用户主页；邮箱注册或登录           |
| 注册用户 | 发布帖子；管理本人帖子；评论、回复与举报；查看回复通知 |
| 管理员   | 删除帖子和评论；封禁用户；查看最小管理页面             |

版主、多级管理员和复杂权限系统延期。

## 4. MVP 功能范围

### 4.1 社区首页

- Header、Sidebar、简单静态 Hero、分类标签和帖子列表；
- 赚钱机会、副业项目、项目验证、项目拆解、获客与变现、收入案例等第一层内容优先；
- Vibe Coding、Agent、RAG、MCP、FDE 和工具资源作为第二层分类；
- 浅色、深色与跟随系统主题；
- 桌面、平板和移动端基础适配；
- Phase 2 使用明确标注的官方 Mock 数据。

### 4.2 邮箱认证

- 邮箱注册；
- 邮箱登录；
- 退出；
- 当前用户查询；
- 简单个人资料。

邮箱验证、密码找回和会话方案在 Phase 3 实现前确定。

### 4.3 帖子

- 浏览帖子列表与详情；
- 发布帖子；
- 作者编辑、删除自己的帖子；
- 使用简单文本输入，不引入复杂富文本编辑器；
- 复杂板块层级和图片对象存储延期。

帖子应允许围绕机会、验证、拆解、变现、收入与失败进行讨论。当前数据模型尚未提供正式帖子类型或收入证明字段，不得通过 Mock 数据伪装这些能力已经实现。

### 4.4 评论与回复

- 一级评论；
- 对评论进行一层回复；
- 作者可删除自己的评论；
- 本轮不实现深层嵌套讨论。

### 4.5 回复通知

- 帖子收到评论时通知作者；
- 评论收到回复时通知被回复用户；
- 查看基础未读/已读状态；
- 点赞、关注等通知延期。

### 4.6 最小治理

- 用户举报内容；
- 管理员删除帖子与评论；
- 管理员封禁用户；
- 最小管理页面；
- 保留必要管理日志。

完整审核工作台、版主管理和复杂处置流程延期。

### 4.7 运行

- 真实 PostgreSQL 与 migration；
- 基础部署、应用日志和数据库备份；
- 邀请 10–50 名目标用户进行小范围测试。

## 5. 延期范围

- Hero 像素小游戏；
- 点赞、收藏、关注和浏览历史；
- 私信、排行榜、积分、徽章；
- 第三方登录；
- AI 自动回答、推荐、搜索、RAG 和 Agent；
- Redis 和完整对象存储体系；
- 复杂富文本编辑器和复杂板块层级；
- 完整审核工作台、高级 SEO 和原生 App。

## 6. 非功能要求

- TypeScript 保持严格模式；
- 前后端与共享包保持既有 Phase 1 边界；
- 环境变量在启动时校验；
- 用户输入在服务端验证并安全输出；
- 数据库不可用时明确报告，不静默忽略；
- 认证、写操作和管理操作需要日志与最小审计；
- 主题和响应式故障不得破坏内容浏览；
- 不采集验证链以外的不必要用户数据。

## 7. Validation MVP 完成定义

1. 访客可浏览真实公开帖子；
2. 新用户可通过邮箱注册、登录和退出；
3. 用户可发布、编辑和删除自己的帖子；
4. 第二位用户可评论并回复；
5. 发帖或评论用户可收到回复通知；
6. 用户可举报内容；
7. 管理员可删除内容并封禁用户；
8. 核心链具有自动化测试和基础日志；
9. PostgreSQL migration、部署和备份流程得到真实验证；
10. 10–50 名测试用户的小范围验证可开始执行。

点赞、收藏、关注、AI 或 Hero 游戏不属于完成条件。

## 8. 当前阶段

Phase 2 只实现静态社区首页 UI，不实现认证、帖子、评论、通知或治理后端。Mock 内容必须明确标识，不得声称为真实社区数据。

## 9. 重新定位说明（2026-07-14）

历史阶段状态和已完成能力保持不变。后续 MVP 内容验证优先面向想利用 AI 做副业的普通用户，其次是独立开发者、AI 创业者和 AI 工程师。不得把技术帖子数量当作定位验证成功；应观察用户是否围绕机会、验证、变现和复盘产生真实内容与回访。
