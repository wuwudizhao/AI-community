# Liftoff 项目目的与范围

> 项目中文名：起飞社区  
> 文档状态：正式定位  
> 生效日期：2026-07-14  
> Slogan：Find. Build. Earn. Repeat.

本文档定义 Liftoff 的正式目的、用户、产品边界与内容优先级。完整定位以 [PRODUCT_VISION.md](PRODUCT_VISION.md) 为真相源。本文描述目标状态，不代表相关功能已经完成。

## 1. 项目目的

Liftoff 是面向普通人、独立开发者、AI 创业者和 AI 工程师的 **AI 创业 / AI 副业赚钱社区**。

项目希望帮助用户通过 AI：

```text
发现机会 → 验证项目 → 做出产品 → 获得收入 → 分享经验
```

Liftoff 的目标不是讨论 AI 本身。Coding 是实现商业目标的一种工具，创造真实收入才是社区最终关注的结果。

## 2. 项目背景

AI 降低了写代码、制作内容、自动化流程和交付服务的门槛，但“能做出来”不等于“有人愿意付费”。普通用户和独立开发者仍然缺少：

- 值得验证的具体机会；
- 判断需求真伪的方法；
- 从想法到最小产品的拆解；
- 获客、定价、销售和交付经验；
- 包含成本、周期与限制的真实收入案例；
- 对失败项目和无效渠道的公开复盘；
- 为商业目标服务的 AI Coding、Agent、RAG 和自动化知识。

Liftoff 将这些经验组织成可浏览、讨论、验证和长期沉淀的社区内容。

## 3. Mission

帮助更多普通人发现机会、验证项目、赚到第一笔收入，最后成长为能够持续构建和交付产品的 AI Builder。

## 4. 目标用户

用户优先级必须按以下顺序理解：

1. **想利用 AI 做副业的人：**需要低门槛机会、验证路径、项目拆解与变现方法；
2. **独立开发者：**希望把开发能力变成产品、服务或持续收入；
3. **AI 创业者：**正在验证市场、商业模式、获客渠道和交付流程；
4. **AI 工程师：**提供并学习 Vibe Coding、Agent、RAG、MCP、FDE 等实现能力。

技术能力不再决定用户是否属于 Liftoff。社区必须让非工程背景用户也能理解、参与并采取行动。

## 5. 用户核心需求

1. 发现值得进一步调查的 AI 赚钱机会；
2. 判断一个副业或产品想法是否值得投入；
3. 获取从需求、制作到发布的项目拆解；
4. 学习定价、获客、成交、交付和复购；
5. 查看具有条件说明的真实收入案例；
6. 了解项目失败、无效渠道和停止条件；
7. 使用 AI Coding、Agent、RAG、MCP 等工具完成项目；
8. 发布自己的项目、结果和问题，并获得反馈；
9. 通过评论、通知和回访形成持续交流。

## 6. 产品定位

Liftoff 是公开社区论坛，围绕“如何利用 AI 创造收入”组织内容。

它不是：

- 以技术新闻和框架讨论为核心的 AI 开发者论坛；
- 收益保证、投资建议或快速致富平台；
- 自动赚钱 Agent；
- 课程商城、软件外包平台或完整即时通讯产品；
- 独立小游戏网站。

## 7. 社区增长闭环

```text
赚钱机会
→ 项目验证
→ 项目拆解
→ AI Coding
→ 技术优化
→ 获客与变现
→ 商业复盘
→ 新的赚钱机会
```

产品、设计和 Roadmap 应优先补齐这条闭环，不能再次退化为 Coding → Coding → Coding 的技术内容循环。

## 8. 内容板块

### 8.1 第一层：赚钱

- 赚钱机会
- 副业项目
- 项目验证
- 项目拆解
- 获客与变现
- 收入案例
- 失败复盘
- 一人公司

### 8.2 第二层：技术

- Vibe Coding
- Agent
- RAG
- MCP
- FDE
- OPC
- 工具与资源

赚钱分类应优先出现在首页导航、Hero 后分类与默认 Feed 中。技术分类用于回答“怎样做出来和优化”，不是首页核心。

## 9. 首页方向

首页继续采用 Header、Sidebar、Hero、分类标签与真实内容 Feed 的现代社区结构，但信息层级调整为：

```text
发现赚钱机会
→ 验证项目
→ 做出来
→ 赚钱
→ 分享经验
```

Hero 建议文案：

> 用 AI 做产品，赚到第一桶金。  
> 发现赚钱机会、验证项目、分享实战经验，学习真正能赚钱的 AI 技术。

原 `Build. Ship. Repeat.` 不再作为正式 Slogan。新 Slogan 为 `Find. Build. Earn. Repeat.`。

## 10. MVP 主链

现有社区行为主链继续有效：

```text
访问首页 → 浏览真实内容 → 注册/登录 → 发布 → 回复 → 通知 → 回访
```

内容验证则新增商业导向：用户是否愿意浏览、发布并讨论真实机会、项目验证、变现过程、收入结果和失败复盘。

## 11. 当前能力与非目标

现有认证、帖子、评论和最小通知是社区基础设施，不因重新定位而回退。本次文档调整不表示首页分类、推荐算法、收入字段或审核功能已经实现。

当前不承诺：

- 收入真实性自动验证；
- 个性化赚钱推荐；
- 项目收益预测；
- 自动生成商业项目；
- 付费课程与交易市场；
- 投资、财务、税务或法律建议；
- Hero 完整游戏。

## 12. 可信内容原则

1. 不使用虚假账号、帖子、收入截图或互动数据；
2. 不把个案包装为普遍可复制结果；
3. 鼓励收入案例说明成本、周期、渠道、口径和限制；
4. 失败项目与没有收入的结果具有独立价值；
5. AI 生成内容必须明确标识；
6. 不使用保证收益、稳赚或制造焦虑的表达；
7. 所有发布功能必须配套必要的举报、审核和处置能力。

## 13. 成功指标

除注册、发帖、回复和回访外，后续应验证：

- 赚钱机会内容的有效阅读与收藏意愿；
- 用户提交项目验证结果的比例；
- 项目拆解引发的实际行动和后续复盘；
- 收入案例的信息完整度与可信反馈；
- 失败复盘的复用价值；
- 技术内容是否能连接到明确项目或商业目标。

具体统计口径、隐私边界和目标值待确认，不得虚构历史数据。

## 14. 当前结论

从 2026-07-14 起，Liftoff 唯一正式定位为 **AI 创业 / AI 副业赚钱社区**。历史“AI Builder / AI 开发者社区”定位已被取代；其技术内容保留为第二层支撑能力。

本次只完成文档收口。代码、UI、API、数据库与现有业务主链保持不变，后续实现必须另行立项与验收。
