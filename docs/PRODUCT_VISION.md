# Liftoff 产品愿景

> 文档状态：正式定位  
> 生效日期：2026-07-14  
> 中文名称：起飞社区  
> Slogan：Find. Build. Earn. Repeat.

## 1. 唯一正式定位

Liftoff 是一个面向普通人、独立开发者和创业者的 **AI 创业 / AI 副业赚钱社区**。

Liftoff 的目标不是泛泛讨论 AI，也不是把 Coding 本身当作终点，而是帮助用户利用 AI 发现机会、验证项目、做出产品、获得收入，并把真实经验继续分享给社区。

> Coding 是工具。创造收入才是最终目标。

技术讨论仍然重要，但属于第二层能力，用于支撑机会验证、产品交付、获客变现和商业复盘。

## 2. Mission

帮助更多普通人：

1. 发现可执行的 AI 赚钱机会；
2. 用低成本方式验证项目；
3. 做出能够交付或销售的产品；
4. 赚到第一笔真实收入；
5. 分享结果并逐步成长为 AI Builder。

## 3. 社区增长闭环

```text
赚钱机会
→ 项目验证
→ 项目拆解
→ AI Coding
→ 技术优化
→ 获客与变现
→ 商业复盘
→ 新的赚钱机会
```

首页、分类、内容推荐和未来功能应围绕这条闭环组织，而不是形成只讨论 Coding、Agent 或模型工具的技术信息流。

## 4. 目标用户优先级

### 第一优先：想利用 AI 做副业的人

包括没有完整技术背景、希望用 AI 尝试副业、服务、小产品或内容生意的普通用户。他们最需要机会判断、验证路径、可执行拆解、获客方式和真实收益经验。

### 第二优先：独立开发者

希望把技术能力转化为可销售产品、订阅服务、自动化方案或客户交付的个人开发者。

### 第三优先：AI 创业者

正在寻找市场机会、验证商业模式、获取用户并建设 AI 原生业务的创业者和小团队。

### 第四优先：AI 工程师

Vibe Coding、Agent、RAG、MCP、FDE 等技术从业者。他们既提供技术供给，也可以学习如何把技术能力转化为收入，但不再是首页信息架构的第一优先用户。

## 5. 核心价值

- **发现机会：**汇集可以进一步验证的 AI 副业、服务和产品机会。
- **降低试错成本：**分享需求判断、验证步骤、投入边界和停止条件。
- **从想法到收入：**连接项目拆解、制作、发布、获客、定价和变现全过程。
- **真实证据：**鼓励披露背景、方法、成本、收入口径、时间范围和限制条件。
- **复盘失败：**记录没有赚到钱的项目、错误假设和无效渠道，防止重复踩坑。
- **技术服务商业：**让 Coding、Agent、RAG、MCP 和 FDE 回到解决具体商业问题的位置。

### 第一层：赚钱与商业实践

4. 项目拆解
5. 获客与变现

## 7. 首页信息架构

```text
Header：品牌、搜索、发布、通知、身份
Sidebar：赚钱分类优先，技术分类次级
Hero：发现机会 → 验证项目 → 做出来 → 赚钱 → 分享经验
Primary Tabs：赚钱机会、副业项目、项目验证、项目拆解、获客与变现、收入案例
Secondary Topics：Vibe Coding、Agent、RAG、MCP、FDE、工具与资源
Feed：真实商业案例与项目实践优先，技术内容作为实现支撑
```

Hero 核心表达：

> 用 AI 做产品，赚到第一桶金。  
> 发现赚钱机会、验证项目、分享实战经验，学习真正能赚钱的 AI 技术。

## 8. 内容排序原则

默认信息优先级为：

```text
真实收入案例
→ 赚钱机会
→ 副业项目
→ 项目验证
→ 项目拆解
→ 获客与变现
→ 失败复盘
→ AI Coding 与技术优化
```

这是一项产品排序原则，不代表当前代码已经具备推荐算法或帖子类型字段。在后端正式支持前，UI 不得用虚假数据假装实现该排序。

## 9. 可信度原则

- 不保证用户一定赚钱，不使用“轻松赚钱”“稳赚”等误导性表达；
- 收入案例必须说明其真实性边界，具体审核规则待确认；
- 收入截图只能作为辅助证据，不能单独证明可复制性；
- 鼓励同时披露投入、成本、周期、渠道、样本和失败条件；
- 不使用虚假用户、虚假项目、虚假收入或 AI 批量内容制造繁荣；
- 涉及投资、金融、税务、法律或平台规则的内容应明确风险与时效性；
- 技术建议和商业案例均应区分事实、个人经验与推测。

## 10. 产品边界

Liftoff 首先仍是公开社区，不是代运营公司、投资顾问、收益承诺平台、课程商城或自动赚钱工具。社区提供信息、经验和协作机会，不对任何项目的收益作保证。

当前已完成的认证、帖子、评论和通知主链继续保留。重新定位不授权修改代码、数据库或 API；后续 UI 与业务调整必须另立实施阶段，并遵守真实数据和兼容现有主链的要求。
