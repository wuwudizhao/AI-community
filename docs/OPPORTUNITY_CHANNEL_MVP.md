# Liftoff Opportunity Channel MVP

日期：2026-07-14

## 目标

将首页的“赚钱机会”入口接入首个真实频道，形成：

```text
首页 → 赚钱机会频道 → 帖子详情 → 评论 / 二级回复 → 通知回访
```

频道不是独立数据模型；第一版只使用现有 `Tag`、`PostTag` 和 `Post`。

## 路由与入口

- 公开频道：`/opportunities`。
- Sidebar、Hero、首页快捷入口和首页 Tab 的“赚钱机会”均连接该路由。
- 发布入口：`/posts/new?tag=赚钱机会`。
- 帖子详情继续使用既有 `/posts/[slug]`。

## 数据边界

频道请求为：

```text
GET /api/posts?tag=赚钱机会&page=1&pageSize=12&sort=latest
```

服务端使用现有标签规范化逻辑把查询值转换为稳定 slug，并通过 `PostTag → Tag.slug` 在数据库内精确筛选。筛选只返回 `PUBLISHED` 帖子，且发生在分页前；评论数仍来自现有已发布评论计数。未提供 `tag` 时，原有列表行为不变。

第一版不显示或推断收入、成本、付费人、评分、来源状态、验证状态或任何不在数据库中的商业字段。

## 发布体验

机会发布页仅自动预填 `赚钱机会` Tag，并提供五项非强制提示：机会、潜在付费人、当前时机、最小验证动作、风险与来源。提交仍采用现有 `title`、`contentMarkdown`、`tags` 契约；普通 `/posts/new` 不受影响。

## 空状态

频道为空时显示：

- 标题：还没有赚钱机会
- 说明：分享你发现的 AI 商业机会，帮助大家判断它是否值得验证。
- 操作：发布第一个赚钱机会

不会填充示例帖子或虚假互动数据。

## 明确延期

其他频道、PostType、商业资料字段、搜索、推荐、点赞、收藏、抓取和游戏逻辑继续延期。认证、Session、评论、二级回复、通知、Prisma schema 和 migration 在本轮不修改。
